# Release notes — v1.3.10-audit-hardening

## v1.3.10-audit-hardening

- `release_check.py` больше не запускает один и тот же combined smoke-suite дважды: `smoke_test.py` остаётся каноническим исполняемым входом, а `smoke_test_combined.py` проверяется как entrypoint-контракт.
- `release_check.py` получил bounded subprocess runner с таймаутами и heartbeat-выводом для долгих проверок; performance/prod/dnd gates запускаются до тяжёлого smoke-corpus.
- `performance_check.py` получил переиспользуемый `startup_import_probe_code()` и timeout, чтобы startup performance probe не мог зависнуть бесконечно.
- `MedicalDocumentService` теперь заранее отвергает неверные расширения первичного документа и ЭПИ, поддерживает `selected_docs="primary"`, корректно обрабатывает `output_dir=None` и не мутирует переданный `override_data`.
- Медицинские документы и дневники теперь отвергают дату выписки раньше даты поступления.
- Медицинский `parse_date()` получил sane bounds 1900–2200, синхронно с дневниковым парсером.
- Drag-and-drop TXT-классификатор теперь читает `utf-8-sig`/`utf-8`/`cp1251`, поэтому cp1251-файл `ЭПИ.txt` не теряется при перетаскивании.
- Дневниковый batch заранее валидирует DOCX/DOCM-файлы, не создаёт папку из одних пробелов в `output_dir`, а повторяющиеся статусы из нескольких файлов схлопывает.
- `_medical_override_data()` нормализует ручную компактную дату поступления перед передачей в renderer.
- Добавлены regression-проверки на все перечисленные сценарии.

# Release notes — v1.3.9-audit-hardening

## v1.3.9-audit-hardening

- Усилен `MedicalDocumentService`: проверяет существование входных файлов, неизвестные типы документов, пустой список выбора и некорректную дату выписки до рендера DOCX.
