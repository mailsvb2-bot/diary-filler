# Медицинский автозаполнитель

Локальная Windows-программа для заполнения медицинских документов и дневников наблюдения в Word `.docx`.

## Что умеет

- читает первичный документ пациента: направление на госпитализацию или первичный осмотр;
- создаёт выбранные документы: первичный осмотр, выписной эпикриз, совместный осмотр, ВК на МСЭ, ВК больничный, акт для РВК;
- заполняет дневники наблюдения с сохранением подписей и строк выписки;
- поддерживает справочник МКБ-10, раздел F, с поиском по коду и названию;
- автоматически меняет мужской/женский род в документах и дневниках по ФИО из первичного документа;
- убирает ЭПИ из итоговых документов, если ЭПИ-файл не выбран;
- умеет сохранить документы без печати или сразу отправить созданные файлы на выбранный принтер;
- сохраняет выбранный принтер между запусками.

## Важно про шаблоны

Для обычного локального запуска рядом с программой может лежать папка `templates`.
Для GitHub Actions и one-file EXE шаблоны также встроены в `src/embedded_templates.py`, поэтому сборка EXE не требует отдельно загружать бинарные `.docx`-шаблоны в репозиторий.

## Как собрать готовый `.exe` через GitHub Actions

1. Откройте вкладку **Actions**.
2. Выберите workflow **Build Windows EXE**.
3. Нажмите **Run workflow**.
4. После завершения откройте последний запуск.
5. Скачайте artifact **MedicalDiaryAutofill-Windows-EXE**.
6. Внутри будет готовый файл:

```text
MedicalDiaryAutofill.exe
```

Конечному пользователю не нужен Python, pip или зависимости. Он запускает `.exe` двойным кликом.

## Локальная сборка на Windows

```bat
СОБРАТЬ_ONEFILE_EXE.bat
```

## Локальный запуск из Python

```bat
pip install -r requirements.txt
python src\main.py
```
